# 补充参考文献工具

## 触发条件

运行 `check_markdown.js` 后，若 `thesis.check.log` 显示参考文献定义（`[^N]:` 行）数量 **< 15**，建议使用本工具补充引用。

## 前置检查

首先确认当前环境是否具备 **web-search 能力**（即是否可调用搜索工具检索外部信息）：

- **有**：继续执行后续步骤。
- **无**：告知用户"当前环境不支持 web search，无法自动调研文献，请手动查阅并补充 `thesis.md` 末尾的参考文献定义"，终止流程。

## 引用质量约束

搜索与筛选文献时严格遵守以下规则：

| 约束 | 要求 |
|------|------|
| 时效性 | 仅考虑 **近三年内**（发表年份 ≥ 当前年份 − 2）发表的文献 |
| 外文比例 | 最终补充的文献中，**外文期刊/会议论文 > 50%** |
| 优先级 | 正式发表 > 预印本；外文期刊 > 中文期刊 > 学位论文/报告 |
| 数量目标 | 补充后总引用数达到 **≥ 15**，上限建议 20 |
| 真实性 | 所有文献必须经过验证（见"验证步骤"） |

## 执行步骤

### Step 1：收集论文中的潜在引用点

读取 `thesis.md`，逐节扫描以下内容，记录每处的**章节位置**和**核心技术/概念关键词**：

- 技术名词首次出现（如"向量数据库"、"语义搜索"、"大语言模型"）
- 数据或性能声明（如"准确率达到 X%"、"延迟低于 Y ms"）
- 设计决策的理论依据（如"采用 TF-IDF 的原因"、"余弦相似度的选择"）
- 已有 `[^N]` 引用但缺少对应定义的位置

为每个引用点生成 1-3 个英文搜索关键词组合，优先使用领域术语。

### Step 2：web search 调研

对每个关键词组合执行搜索，目标来源优先级：

1. **arXiv / ACL Anthology / IEEE Xplore / ACM DL / Springer** — 正式发表的期刊/会议论文
2. **Google Scholar** — 覆盖面广
3. **CNKI / 万方** — 中文文献补充（控制比例 < 50%）

每次搜索记录候选文献的：标题、作者、来源、年份、DOI/URL。

### Step 3：格式化引用

按 **GB/T 7714-2015** 格式整理，常用类型示例：

```
# 期刊论文（最常用）
[N] 作者1, 作者2, 作者3, 等. 题名[J]. 期刊名, 年份, 卷(期): 起止页.

# 会议论文
[N] 作者1, 作者2. 题名[C]// 会议名. 出版地: 出版者, 年份: 页码.

# 网络资源（技术文档/博客慎用）
[N] 责任者. 题名[EB/OL]. (更新日期)[引用日期]. URL.
```

作者规则：≤3 人全列；>3 人列前三后加"等"（中文）或"et al."（英文）。姓在前，名在后。

### Step 4：验证引用真实性

调研完成后，**启动独立 sub-agent** 执行验证（若当前系统支持 Task/sub-agent 能力）：

```
验证任务：逐条检查以下文献是否真实存在。
对每条文献：
1. 用标题 + 作者 web search 确认可检索到
2. 确认发表年份、期刊/会议名称与记录一致
3. 对无法确认的文献标记 [UNVERIFIED]，并建议删除或替换

待验证文献列表：
[粘贴 Step 3 生成的完整列表]
```

若系统**不支持 sub-agent**，在同一会话中对每条文献单独 web search 验证，移除 `[UNVERIFIED]` 条目后再写入论文。

### Step 5：写入 thesis.md

在 `thesis.md` 末尾的参考文献脚注区，按引用在正文中**首次出现的顺序**追加：

```markdown
[^N]: 作者. 题名[J]. 期刊名, 年份, 卷(期): 页码.
```

同时在正文对应位置补充行内引用标记 `[^N]`（上角标）。

写入后重新运行检查
确认无 `reference.count_low` 警告。
